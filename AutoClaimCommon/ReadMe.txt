This sample uses 6 projects: AutoClaimCommon (library), AutoClaimNoSQL (library), AutoClaimSQL (library), AutoClaim (console application), AutoClaimWebService (Wep Api service) and AutoClaimWebClient (HttpClient console application)/


Setup:

Uses Local SQL Database and VelocityDB (NoSQL) databases.

Enable NuGet packages restore since packackes are not included in download (just configured) 


Libraries:

AutoClaimCommon - contains an interface class IManageAutoClaim used by non web api implementations and AutoClaim console application. Also contains a BAT file for generating C# classes from MitchellClaim.xsd.

AutoClaimNoSQL - a NoSQL implementation using VelocityDB for storing, reading, updating and deleting claims via AutoClaim console application.

AutoClaimSQL - an Entify Framework code first implementation for storing, reading, updating and deleting claims in a local SQL database via AutoClaim console application.


Applications:

AutoClaim - a console application that by default uses the NoSQL (VelocityDB) database for managing claims. Add any to command line parameter to make it use local SQL database via Entity Framwork instead.

AutoClaimWebService - a web api server using Entity Framework to manage claims.

AutoClaimWebClient - a web api client for testing AutoClaimWebService. Not all operations fully implemented. Adding new claims, reading claim and all claims works.


Input data files:

MitchellClaim.xsd - XML schema definition for auto claims.

create-claim.xml – Sample XML used to create a claim. 

update-claim.xml - updates to an auto claim (only contains updated data - not original values except claim number)


Generated files (and then updated):

MitchellClaim.cs - generated by "xsd MitchellClaim.xsd /classes" and then updated adding primary keys as required by Entity Framework.

MitchellClaimPlus.cs - generated by "xsd MitchellClaim.xsd /classes" and then updated for optimal use with VelocityDB (NoSQL); use OptimizedPersistable as base class, make most data nullable (so we can tell what is updated), rename field names with m_ prefix (better naming), add Unpersist api, and put it in namespace AutoClaimUsingNoSQL.
